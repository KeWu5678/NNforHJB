# Sparse HJB Control

This context describes the optimal-control language used by the project when generating HJB value-function data and fitting sparse neural representations.

## Language

**Open-Loop Problem**:
An optimal-control solve for one fixed initial state that returns an optimal trajectory, adjoint trajectory, control, value, and value-gradient sample.
_Avoid_: Offline control, trajectory-only solve

**Fixed-Target Value**:
The smooth value function obtained by fixing one terminal target in a multi-target control problem. A multi-target value is the pointwise minimum of two or more fixed-target values.
_Avoid_: Branch, mode, class, label

**Policy Transformation**:
An exact symmetry relation that maps a value or policy for one fixed-target problem and transformed state to the corresponding value or policy for another fixed-target problem.
_Avoid_: Data augmentation, heuristic symmetry

**Phase-Capture Value**:
The value function for driving an oscillator toward one of several acceptable phase targets while paying running control and state costs.
_Avoid_: Regulator value

**Pendulum Swing-Up Value**:
The paper-backed open-loop example from Han and Yang, arXiv:2312.17467. The
system is an inverted pendulum with state `(theta, omega)` and motor torque
control. The underlying problem minimizes the integral of
`q1(2 - 2 cos(theta)) + q2 omega^2 + R u^2` subject to the pendulum ODE.
Samples are generated by starting on a small local-LQR level set near the
upright equilibrium and integrating the PMP system backward. The returned
costate is the value gradient.
_Avoid_: Treating it as a generic finite-horizon terminal-target problem

**Backward-PMP Sample**:
An open-loop value-gradient sample produced by reversing an optimal trajectory
from the local LQR region. It records `(x, V(x), dV(x))` without solving a new
finite-dimensional control optimization at every grid point.
_Avoid_: Rollout-only sample

**Run Record**:
The durable structured record for one experiment run. It captures the run
configuration, status, timing, metrics, and artifact paths needed to reproduce
or compare the run.
_Avoid_: Treating console progress or free-form text logs as the experiment
source of truth

**Console Progress**:
Human-facing status emitted while a script is running. It helps monitor a run
but is not the authoritative experiment record.
_Avoid_: Experiment log, result log

**Run Artifact**:
A file produced by a run, such as a plot, checkpoint, dataset, summary table,
or diagnostic text log. Artifacts support the run record but do not replace it.
_Avoid_: Calling every generated file a log

## Example Selection Notes

The modified Van der Pol phase-capture construction is not the preferred
example for the nonsmooth open-loop study. Its switching surface is created by
an artificial minimum over two terminal targets, so it is useful for optimizer
debugging but weak as a paper-backed HJB example.

The pendulum swing-up example is stronger because the nonsmoothness comes from
the actual periodic optimal-control geometry: clockwise and counter-clockwise
swing-up branches meet across a curved switching set. That is the geometry
needed when discussing semiconcavity, discontinuous gradients, and the failure
mode for overly smooth value approximators.

## Example Dialogue

Dev: "Should I train on one fixed-target value or on the phase-capture value?"

Domain expert: "Train on the phase-capture value if you want the switching surface; train on a fixed-target value if you want a smooth target."

Dev: "Can I reuse a solve for the positive target?"

Domain expert: "Yes, when the Van der Pol target set is symmetric. The policy transformation maps the positive fixed-target policy at the reflected state to the negative fixed-target policy at the original state."
