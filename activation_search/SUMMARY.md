## Definitions of top activations

```python
# smoothy_relu_w0_25: C¹ piecewise — flat for x<-w, quadratic for |x|<w, linear for x>w
def smoothy_relu(x, w=0.25):
    return torch.where(x>=w, x-w/2,
           torch.where(x<=-w, 0., (1/(2*w))*(x+w)**2))

# gelu_b0_25: gelu with width parameter scaled to 0.25
def gelu_b0_25(x):
    return 0.5 * x * (1 + torch.erf(0.25 * x / sqrt(2)))

# softplus_b0_25: softplus with beta=0.25
def softplus_b0_25(x):
    return F.softplus(x, beta=0.25)

# qr_0_1: x + 0.1*x^2 (near-quadratic; effectively a polynomial-feature net)
def qr_0_1(x):
    return x + 0.1 * x**2
```

## Key findings

1. **Smoothness ↔ sparsity is a fundamental tradeoff.** Sharper activations fit better
   but need many neurons; smoother ones need few but cap at higher H1.

2. **β≈0.25 is a robust sweet spot** across three independent activation families
   (gelu, swish/silu, softplus). Rescaling beta is mathematically equivalent to
   rescaling inner weights, so the empirical effect is partly an optimization
   artifact (algorithm's preferred weight-norm regime), not a fundamental property.

3. **smoothy_relu_w0_25 is the only activation achieving h1<0.18** but with high
   seed variance (h1 ranges 0.054-0.230 over 10 seeds, std=7.12 on score).

4. **The score metric `h1·n` is biased toward low-rank activations** like qr_*, sp2_*.
   These cap at h1≈0.36 because their function space is essentially polynomial.
   Filter to h1<0.30 for meaningful sparse fits.

5. **Bounded activations (tanh, sigmoid, softsign, atan) underperform** — high
   neuron count and high h1.

6. **Radial activations (matern52, gaussian) are worst** — 140+ neurons.

## Methodology

- Per (activation, seed): full PDPA_v2 sweep over 5 gammas;
  `best_score = h1[best_iter] * best_neurons` at the best gamma.
- 5 seeds per activation (42–46); top candidates re-validated on 10 seeds.
- All algorithm settings frozen at notebook defaults: alpha=1e-5, num_iterations=10,
  num_insertion=50, pruning_threshold=1e-5.
